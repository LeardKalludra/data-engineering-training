# Day 4 - Python and SQL Logic Review

## Goal of the Practice
The goal of this assignment is to understand and compare data processing workflows across Python and SQL architectures. By processing an identical order tracking dataset using both environments, we examine how structural operations like record filtering, iterative math calculations, sorting configurations, and conditional summaries map conceptually across memory environments and relational engine scripts.

## Files Included
- `python_orders.py` - Complete logic scripts running the programmatic assignments inside defined processing functions.
- `setup.sql` - Structural database setup query building the target database layout and populating it with sample mock transaction records.
- `sql_queries.sql` - Dedicated evaluation queries handling structural tasks, filtration checks, sorting requests, and relational views.
- `comparison.md` - Analysis summary breaking down processing differences and operational paradigms between procedural statements and declarative rules.
- `daily_report.txt` - Tracking document detailing personal technical notes and daily execution summaries.
